# Nextern

## Project Vision

Nextern is a mobile application designed to provide learners with easy access to learning programs while giving administrators tools to monitor programs, assignments, learner progress, announcements, and feedback through a simple and user-friendly interface.

📄 For full project details, see our [App Proposal Document](https://github.com/Jonathan-Go4th/Project-Nextern/blob/main/Nextern%20-%20App%20Proposal.pdf) and [Week 2 Deliverables](https://github.com/Jonathan-Go4th/Project-Nextern/blob/main/Nextern%20-%20Deliverables%20Week%202.pdf).

## Objectives

- Provide learners with access to available learning programs and internships.
- Enable learners to monitor their learning progress.
- Allow learners to submit assignments and access certificates.
- Help administrators monitor learners, submissions, announcements, and feedback.
- Create a consistent mobile experience for learners and administrators.

## Navigation Flow

### Learner Flow

1. **Login / Sign Up** → Log in or create a learner account.
2. **Dashboard (Home)** → View course-completion progress, the current module, and saved programs.
3. **Browse** → Explore programs and internships using search and category filters.
4. **Tasks** → View and submit assignments, track submission status, and access certificates.
5. **Profile** → View account information and log out.

The bottom navigation bar—Home, Browse, Tasks, and Profile—allows learners to move freely between these sections.

### Administrator Flow

1. **Admin Login** → Access the Admin Login screen using the **Login as Admin** option on the learner login page.
2. **Admin Dashboard** → View active users, average program completion, quick actions, and recent submissions.
3. **Users** → Monitor learner progress and review feedback or support-related comments.
4. **Updates** → Create and manage announcements for learners.
5. **Settings** → Reserved for future administrative configuration.

The bottom navigation bar—Dashboard, Users, Updates, and Settings—follows the same navigation pattern as the learner interface.

## Student Features

- **Login** – Account creation using name, email, password, and acceptance of the terms and conditions; email-password login; password recovery; Google sign-in; and a separate Admin Login entry point.
- **Home** – Course-completion percentage, current module, Resume Learning action, and saved programs.
- **Browse** – Search and filter internships or learning programs by category, mode, and type; save opportunities and access program details.
- **Tasks** – View assignments with due, submitted, and under-review statuses; submit assignments; and download certificates.
- **Profile** – View learner account information and log out.
